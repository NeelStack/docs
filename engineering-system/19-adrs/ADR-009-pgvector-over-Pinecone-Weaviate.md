# ADR-009 — pgvector over Pinecone/Weaviate

## Status
**Accepted** — In effect as of 2026-07-04

## Context
DhruvaOS includes AI-native features: semantic Q&A for school handbooks, behavioral predictions, and automated insights. These require storing high-dimensional vector embeddings generated by LLMs.

We evaluated:
1. **Pinecone**: Fully managed, high performance, but vendor lock-in and high monthly cost.
2. **Weaviate / Qdrant**: Dedicated vector databases. High performance but introduces additional infrastructure overhead.
3. **pgvector extension**: Integrates native vector storage directly into PostgreSQL.

## Decision
We chose **pgvector** as our primary vector store. By enabling `pgvector` inside our core PostgreSQL instance, we avoid deploying and managing a separate database engine, keep transactional integrity (ACID) for vector rows, and query vectors via standard SQL.

## Consequences
- **Developer Overhead**: No separate SDKs required; we query vectors via SQLAlchemy.
- **Costs**: 0 additional host costs; pgvector runs inside our PostgreSQL cluster.
