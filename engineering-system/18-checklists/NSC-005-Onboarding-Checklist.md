# NSC-005 — Engineer Onboarding Checklist

This checklist guides new engineers through their first week at NeelStack. The engineering manager and buddy engineer are responsible for completion.

---

## Day 1 — Access & Setup

### Accounts
- [ ] GitHub account added to NeelStack organization
- [ ] AWS IAM credentials provisioned (least-privilege)
- [ ] Slack/communication tools access granted
- [ ] 1Password team vault access
- [ ] Jira/project management access

### Development Environment
- [ ] MacBook provisioned with engineering standard configuration
- [ ] Local development setup completed (Docker Desktop, VS Code, Node.js, Python)
- [ ] SSH key generated and added to GitHub
- [ ] GPG key generated for signed commits
- [ ] Product repositories cloned and running locally

---

## Week 1 — Standards & Culture

### Engineering Constitution (Required Reading)
- [ ] NES-000 — Engineering Handbook (complete read)
- [ ] NES-001 — Company Vision
- [ ] NES-004 — Engineering Culture
- [ ] LAW-001 to LAW-010 — All Engineering Laws
- [ ] TECH-001 — Technology Stack
- [ ] REPO-001 to REPO-008 — Repository Standards

### Architecture Understanding
- [ ] NES-100 — Architecture Principles
- [ ] NES-104 — Clean Architecture
- [ ] NES-103 — Domain-Driven Design
- [ ] Read all relevant ADRs in `19-adrs/`

### First Tasks
- [ ] Fix a "good first issue" tagged bug
- [ ] Submit first PR and go through review process
- [ ] Pair programming session with buddy engineer (minimum 2 hours)

---

## Week 2 — Domain Knowledge

- [ ] Domain walkthrough with tech lead (architecture + codebase)
- [ ] Attended one team standup and sprint planning
- [ ] Understand the team's on-call rotation schedule
- [ ] Set up monitoring dashboards for the team's services
