# Contributing Guidelines

Welcome to the **NeelStack Docs Repository**.

## Document Standard

Every document in the `engineering-system/` directory must adhere to the frozen **Document Standard** layout:

1. Metadata
2. Executive Summary
3. Purpose
4. Scope
5. Audience
6. Background
7. Architecture
8. Standards
9. Rules
10. Decision Rationale
11. Best Practices
12. Anti-patterns
13. Examples
14. AI Context
15. Mermaid Diagrams
16. Compliance Requirements
17. Success Metrics
18. Checklists
19. Related Documents
20. Version History

## Naming & Numbering Convention

Ensure files follow the correct prefixes:
- **NES**: NeelStack Engineering Standard
- **NAP**: NeelStack Architecture Principle
- **LAW**: Engineering Law
- **TECH**: Technology Standard
- **REPO**: Repository Standard
- **ADR**: Architecture Decision Record
- **RFC**: Request for Comments
- **NST**: Template
- **NSC**: Checklist

## Maturity Level

All documents start at **L0 - Draft** and must proceed through the review process to reach **L5 - Gold Standard**.
